4. **배포 설정**:

   - **Publish directory**: 비워둠 (루트 디렉토리)

### 3. 도메인 설정

### 4. 배포 설정 최적화
```yaml
# netlify.toml (선택사항)

  publish = "."

[[headers]]

  [headers.values]

    X-Content-Type-Options = "nosniff"
    Referrer-Policy = "strict-origin-when-cross-origin"

[[redirects]]
  from = "/index.html"
  to = "/"
  status = 301
```

## 🛠️ 트러블슈팅

### 일반적인 문제들

#### 1. LIFF 앱이 로드되지 않음
**증상**: 흰 화면 또는 에러 메시지
**원인**: 
- Endpoint URL 잘못 설정
- HTTPS 미사용
- CORS 정책 위반

**해결방법**:
```bash
# Netlify URL 확인
https://your-app.netlify.app (올바른 형식)
https://your-app.netlify.app/index.html (잘못된 형식)

# LINE Developers Console에서 수정
Endpoint URL: https://your-app.netlify.app
```

#### 2. API 호출 실패
**증상**: "네트워크 오류" 메시지
**원인**:
- n8n 워크플로우 비활성화
- API 엔드포인트 URL 오류
- CORS 설정 문제

**해결방법**:
```javascript
// 브라우저 개발자 도구에서 확인
console.log('API URL:', API_CONFIG.BASE_URL);

// n8n에서 워크플로우 활성화 확인
// Webhook 응답 테스트
```

#### 3. 팀 선택 옵션이 없음
**증상**: 빈 팀 선택 화면
**원인**:
- API 빈 응답
- Fallback 데이터 누락

**해결방법**:
```javascript
// Console 로그 확인
"API returned empty array, using fallback data"

// Fallback 데이터 점검
if (!teams || teams.length === 0) {
    // Fallback 로직 실행
}
```

#### 4. Notion 데이터 저장 실패
**증상**: 데이터가 Notion에 저장되지 않음
**원인**:
- Notion Integration 권한 부족
- 데이터베이스 ID 오류
- 필드 타입 불일치

**해결방법**:
```bash
# Notion Integration 권한 확인
Database → Share → n8n Integration → Can edit

# n8n에서 Notion 노드 설정 확인
Database ID: 정확한 ID 입력
Field Mapping: 올바른 필드 매핑
```
